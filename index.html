Using classes

When we use a class like turtle.Turtle() we call the function __init__ that initializes some space for the instance a new instance.

So for the send text mini-project we had to use the Twilio package to write a program to send ourselves a text message. In this mini-project we have to import a folder from inside the folder Twilio called Twilio.rest to import the class TwilioRestClient and we use this class in our code.

Classes are like a blue print so from the blue print we can create different objects or instances from that blue print.

We know that the Python Standard library comes with different modules or folder like webbrowser, turtle, and time. When we want to use one of these modules we have to import it so that python can know that we are going to call functions or classes from those modules.

Some functions like open are built in so we don't have to import anything because it is readily availible because it is used very often.

Inheritance is a way we can reuse code which is huge and is a big help when programming. It's easier to understand. For example we can create a class called Video's which will be the parent class then we cant have to child classes like the class movies that we have already written and the create a new class Tv-Shows. They will inhereit the things they have in common with the parent like title and run time.

Method is a function which is a member of a class. For example show_info in the Parent class is a method. We can reuse this method in in the child class even though this function is not explicitly defined in the child class we can still use the function because we inherit it from the parent class.
